# セキュリティ仕様書

## 概要

本プロジェクトでは、Web アプリケーションの主要な脅威である「SQL インジェクション」と「クロスサイト・スクリプティング（XSS）」の対策を、以下の仕様に基づき実施しました。

## 1. 🔒 SQL Injection (SQL インジェクション)対策

本アプリでは、Rails が提供する ActiveRecord を使用しており、where をハッシュ形式で記述するなど、プレースホルダ付きの安全なクエリのみを使用しています。
また、Strong Parameters を導入し、許可されたパラメータ以外を受け取らないように実装しています。
このため、文字列連結による生の SQL を使用せず、SQL Injection のリスクを防止しています。

## 2. 🔒 XSS （クロスサイトスクリプティング）対策

Rails はデフォルトで出力時に HTML をエスケープする仕組みがあり、ユーザー入力がそのままスクリプトとして実行されないようになっています。
Next.js（React）側でも dangerouslySetInnerHTML を使用せず、コンポーネントは通常の JSX レンダリングで安全に表示しています。
また、フォーム入力にはバリデーションを設け、不正なデータが保存されないようにしています。
これらにより、XSS（クロスサイトスクリプティング）のリスクを軽減しています。
