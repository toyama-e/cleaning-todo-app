# セキュリティ仕様書

## 概要

本プロジェクトでは、Web アプリケーションの主要な脅威である「SQL インジェクション」と「クロスサイト・スクリプティング（XSS）」の対策を、以下の仕様に基づき実施しました。

## 1. 🔒 XSS （クロスサイト・スクリプティング）対策

### XSS は、悪意のあるスクリプトが Web ページに埋め込まれ、他のユーザーに被害を与えることを防ぐための対策です。

本アプリケーションは Rails を API モードで使用しており、HTML を返さず JSON のみを返します。
そのため、ブラウザでスクリプトが実行される XSS は原理的に発生しません。

## 2. 🔒 SQL Injection (SQL インジェクション)対策

### SQL インジェクションは、データベースへの不正な SQL コマンドの挿入を防ぐための対策です。

本アプリでは、Rails が提供する ActiveRecord を使用しており、where をハッシュ形式で記述するなど、プレースホルダ付きの安全なクエリのみを使用しています。
また、Strong Parameters を導入し、外部から渡された値のうち許可したカラムだけを受け付けるように実装しています。
このため、文字列連結による生の SQL を使用せず、SQL Injection のリスクを防止しています。
